[Railway](https://railway.app) is a deployment platform that makes it easy to deploy applications with minimal configuration. This guide walks you through deploying Vibe Kanban Cloud on Railway.

## Why Railway?

- **Simple setup** - Deploy from GitHub with minimal configuration
- **Managed infrastructure** - Automatic SSL, scaling, and monitoring
- **Built-in PostgreSQL** - One-click database provisioning
- **Transparent pricing** - Pay for what you use

## Prerequisites

- [Railway account](https://railway.app) (GitHub login recommended)
- OAuth credentials from [GitHub](https://github.com/settings/developers) or [Google](https://console.cloud.google.com/apis/credentials)

## Step 1: Fork the Repository

1. Go to [github.com/BloopAI/vibe-kanban](https://github.com/BloopAI/vibe-kanban)
2. Click **Fork** to create your own copy
3. This allows Railway to deploy from your repository

## Step 2: Create Railway Project

1. Go to [Railway Dashboard](https://railway.app/dashboard)
2. Click **New Project**
3. Select **Deploy from GitHub repo**
4. Choose your forked `vibe-kanban` repository

## Step 3: Add PostgreSQL

1. In your Railway project, click **New**
2. Select **Database** → **PostgreSQL**
3. Railway automatically provisions a PostgreSQL instance

## Step 4: Configure the Service

Click on your deployed service and go to **Settings**:

### Build Settings

- **Root Directory**: `crates/remote`
- **Build Command**: Leave empty (uses Dockerfile)
- **Dockerfile Path**: `Dockerfile`

### Environment Variables

Go to the **Variables** tab and add:

```env
# Required
VIBEKANBAN_REMOTE_JWT_SECRET=<generate with: openssl rand -base64 48>
ELECTRIC_ROLE_PASSWORD=<secure_password>

# OAuth (at least one required)
GITHUB_OAUTH_CLIENT_ID=your_github_client_id
GITHUB_OAUTH_CLIENT_SECRET=your_github_client_secret
GOOGLE_OAUTH_CLIENT_ID=your_google_client_id
GOOGLE_OAUTH_CLIENT_SECRET=your_google_client_secret

# URLs (Railway provides these via ${{RAILWAY_PUBLIC_DOMAIN}})
SERVER_PUBLIC_BASE_URL=https://${{RAILWAY_PUBLIC_DOMAIN}}
VITE_APP_BASE_URL=https://${{RAILWAY_PUBLIC_DOMAIN}}
VITE_API_BASE_URL=https://${{RAILWAY_PUBLIC_DOMAIN}}

# Database (Railway auto-provides DATABASE_URL, but we need this format)
SERVER_DATABASE_URL=${{DATABASE_URL}}

# Email (optional)
LOOPS_EMAIL_API_KEY=

# Server config
RUST_LOG=info
SERVER_LISTEN_ADDR=0.0.0.0:${{PORT}}
```

<Info>
Railway automatically provides `DATABASE_URL` when you link the PostgreSQL service. Use the variable reference `${{DATABASE_URL}}` to access it.
</Info>

## Step 5: Link Database

1. Click on your main service
2. Go to **Variables**
3. Click **Add Reference** → Select your PostgreSQL service
4. This makes `DATABASE_URL` available to your app

## Step 6: Deploy ElectricSQL

ElectricSQL is required for real-time sync:

1. Click **New** → **Empty Service**
2. Name it `electric`
3. Go to **Settings**:
   - **Image**: `electricsql/electric:1.3.3`

4. Add environment variables:

```env
# Note: DATABASE_URL must use the electric_sync user, not the default postgres user
# The format is: postgresql://electric_sync:<ELECTRIC_ROLE_PASSWORD>@<postgres-host>:5432/remote?sslmode=disable
# You'll need to construct this manually using the ELECTRIC_ROLE_PASSWORD
DATABASE_URL=postgresql://electric_sync:${ELECTRIC_ROLE_PASSWORD}@${{Postgres.RAILWAY_PRIVATE_DOMAIN}}:5432/remote?sslmode=disable
PG_PROXY_PORT=65432
AUTH_MODE=insecure
ELECTRIC_INSECURE=true
ELECTRIC_MANUAL_TABLE_PUBLISHING=true
ELECTRIC_USAGE_REPORTING=false
ELECTRIC_FEATURE_FLAGS=allow_subqueries,tagged_subqueries
ELECTRIC_ROLE_PASSWORD=<same_as_main_service>
```

<Warning>
ElectricSQL connects using the `electric_sync` database user, which is created by the main remote server on first startup. Deploy the main service first and wait for it to be healthy before ElectricSQL can connect.
</Warning>

5. Link the PostgreSQL database to this service too

## Step 7: Update Main Service with Electric URL

Add to your main service variables:

```env
ELECTRIC_URL=https://${{electric.RAILWAY_PUBLIC_DOMAIN}}
```

## Step 8: Configure OAuth Callbacks

Get your Railway public domain from the service settings.

Update your OAuth applications:

<Tabs>
<Tab title="GitHub">
- **Authorization callback URL**: `https://your-app.up.railway.app/v1/oauth/github/callback`
</Tab>

<Tab title="Google">
- **Authorized redirect URI**: `https://your-app.up.railway.app/v1/oauth/google/callback`
</Tab>
</Tabs>

## Step 9: Deploy

Railway automatically deploys when you push to your repository. You can also trigger a manual deploy:

1. Go to your service
2. Click **Deploy** → **Deploy Now**

<Info>
The first build takes 10-15 minutes. You can watch the build logs in the **Deployments** tab.
</Info>

## Step 10: Generate Domain

1. Go to your main service
2. Click **Settings** → **Networking**
3. Click **Generate Domain**
4. Railway provides a `.up.railway.app` subdomain

## Step 11: Verify Deployment

Open your Railway domain in a browser. You should see the Vibe Kanban login page.

## Connecting Your Client

```bash
export VK_SHARED_API_BASE=https://your-app.up.railway.app
```

## Custom Domain

1. Go to **Settings** → **Networking**
2. Click **Custom Domain**
3. Enter your domain (e.g., `vk.example.com`)
4. Add the CNAME record to your DNS

Update environment variables:

```env
SERVER_PUBLIC_BASE_URL=https://vk.example.com
VITE_APP_BASE_URL=https://vk.example.com
VITE_API_BASE_URL=https://vk.example.com
```

Update OAuth callback URLs to use your custom domain.

## Monitoring

### View Logs

Click on your service and go to **Deployments** → Select a deployment → **Logs**

### Metrics

Railway provides built-in metrics for CPU, memory, and network usage in the **Metrics** tab.

## Scaling

Railway automatically scales based on usage. You can configure limits in **Settings**:

- **Memory**: Increase for better performance
- **vCPU**: Increase for faster builds

## Costs

Railway uses usage-based pricing:

- **Starter plan**: $5/month credit, then pay-as-you-go
- **Pro plan**: $20/month with team features

For a small team, expect approximately $10-20/month depending on usage.

## Troubleshooting

<AccordionGroup>
<Accordion title="Build fails">
Check the build logs for errors. Common issues:
- Insufficient memory during Rust compilation
- Missing environment variables

Try increasing the service memory in Settings.
</Accordion>

<Accordion title="Database connection errors">
Ensure the PostgreSQL service is linked:
1. Go to your service → Variables
2. Verify `DATABASE_URL` reference exists
3. Check PostgreSQL service is running
</Accordion>

<Accordion title="OAuth errors">
Verify callback URLs match exactly:
- Check for trailing slashes
- Ensure HTTPS is used
- Confirm domain matches Railway domain
</Accordion>

<Accordion title="ElectricSQL not connecting">
1. Verify Electric service is running
2. Check `ELECTRIC_URL` in main service
3. Ensure both services are linked to the same PostgreSQL
</Accordion>
</AccordionGroup>

## Updating

Railway automatically deploys when you push to your GitHub repository:

```bash
git pull upstream main  # Get latest from original repo
git push origin main    # Push to your fork
```

Railway detects the push and starts a new deployment.
