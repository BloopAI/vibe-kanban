## Overview

Preview Mode provides an embedded browser experience within Vibe Kanban, allowing you to test and iterate on your web applications without leaving the development environment. This feature eliminates the need to switch between your browser, IDE, and Vibe Kanban by providing live preview capabilities and precise component selection tools.

**Key Benefits:**
- **Embedded viewing**: See your application running directly in Vibe Kanban
- **Precise component selection**: Click to select specific UI components for targeted feedback
- **Seamless workflows**: No context switching between tools
- **Real-time preview**: Instant visual feedback as changes are made

## Prerequisites

Before using Preview Mode, ensure you have:

1. **Development server script configured** for your project
2. **Web Companion installed** in your application (for component selection)
3. **Project running** a web application that serves to localhost

## Setting Up Preview Mode

<Steps>
<Step title="Configure Development Server Script">
  Navigate to your project settings and configure the development server script that starts your local development environment.

  **Common examples:**
  - `npm run dev` (Vite, Next.js)
  - `npm start` (Create React App)
  - `yarn dev` (Yarn projects)
  - `pnpm dev` (PNPM projects)

  </Frame>

  <Tip>
  Ensure your development server prints the URL (e.g., `http://localhost:3000`) to stdout/stderr for automatic detection.
  </Tip>
</Step>

<Step title="Install Web Companion">
  For precise component selection, install the `vibe-kanban-web-companion` package in your application.

  <Info>
  **Recommended**: Use the "Install companion automatically" button in the Preview tab to have Vibe Kanban create a task that installs and configures the companion for you.
  </Info>

  **Manual Installation:**

  Add the dependency to your project:
  ```bash
  npm install vibe-kanban-web-companion
  ```

  Then add the companion to your application:

  <Tabs>
  <Tab title="Create React App">
    Add to your `src/index.js` or `src/index.tsx`:
    ```jsx
    import { VibeKanbanWebCompanion } from 'vibe-kanban-web-companion';
    import React from 'react';
    import ReactDOM from 'react-dom/client';
    import App from './App';

    const root = ReactDOM.createRoot(document.getElementById('root'));
    root.render(
      <React.StrictMode>
        <VibeKanbanWebCompanion />
        <App />
      </React.StrictMode>
    );
    ```
  </Tab>

  <Tab title="Next.js">
    Add to your `pages/_app.js` or `pages/_app.tsx`:
    ```jsx
    import { VibeKanbanWebCompanion } from 'vibe-kanban-web-companion'
    import type { AppProps } from 'next/app'

    function MyApp({ Component, pageProps }: AppProps) {
      return (
        <>
          <VibeKanbanWebCompanion />
          <Component {...pageProps} />
        </>
      )
    }
    ```
  </Tab>

  <Tab title="Vite">
    Add to your `src/main.jsx` or `src/main.tsx`:
    ```jsx
    import { VibeKanbanWebCompanion } from "vibe-kanban-web-companion";
    import React from "react";
    import ReactDOM from "react-dom/client";
    import App from "./App";

    ReactDOM.createRoot(document.getElementById("root")).render(
      <React.StrictMode>
        <VibeKanbanWebCompanion />
        <App />
      </React.StrictMode>
    );
    ```
  </Tab>
  </Tabs>

  <Note>
  The Web Companion is automatically tree-shaken from production builds, so it only runs in development mode.
  </Note>
</Step>

## Using Preview Mode

### Accessing the Preview

Once your development server is running and a URL is detected:

1. **Navigate to Preview tab** in the task details panel
2. **View embedded application** in the iframe
3. **Interact with your app** directly within Vibe Kanban

### Component Selection

When the Web Companion is installed, you can precisely select UI components for targeted feedback:

<Steps>
<Step title="Activate Selection Mode">
  Use one of these methods to select components:
  - **Alt + Click**: Hold Alt and click on any component
  - **Context Menu**: Right-click and select "Edit in Vibe Kanban"

  <Frame>
  {/* Placeholder for component selection screenshot */}
  <img src="/images/preview-mode-component-selection.png" alt="Component selection interface showing selectable elements highlighted" />
  </Frame>
</Step>

<Step title="Choose Component Depth">
  When you click a component, Vibe Kanban shows a hierarchy of components from innermost to outermost. Select the appropriate level for your feedback:

  - **Inner components**: For specific UI elements (buttons, inputs)
  - **Outer components**: For broader sections (cards, layouts)

  <Frame>
  {/* Placeholder for component depth selection screenshot */}
  <img src="/images/preview-mode-component-depth.png" alt="Component depth selection showing hierarchy of selectable components" />
  </Frame>
</Step>

<Step title="Provide Targeted Feedback">
  After selecting a component, write your follow-up message. The coding agent will receive:
  - **Precise DOM selector** information
  - **Component hierarchy** and source file locations
  - **Your specific instructions** about what to change

  <Check>
  No need to describe "the button in the top right" - the agent knows exactly which component you mean!
  </Check>
</Step>
</Steps>

## Troubleshooting

### URL Not Detected

If the preview doesn't load automatically:

<AccordionGroup>
<Accordion title="Check Development Server Logs">
  1. Navigate to the **Processes** tab
  2. Look for your development server process
  3. Verify the server started successfully
  4. Ensure a URL like `http://localhost:3000` appears in the logs

  <Warning>
  The development server must print a URL to stdout/stderr for automatic detection.
  </Warning>
</Accordion>

<Accordion title="Verify URL Format">
  Supported URL formats include:
  - `http://localhost:3000`
  - `https://localhost:3000`
  - `http://127.0.0.1:3000`
  - `http://0.0.0.0:5173`

  URLs using `0.0.0.0` or `::` are automatically converted to `localhost` for embedding.
</Accordion>

<Accordion title="Manual URL Entry">
  If automatic detection fails, you can:
  1. Copy the URL from your development server logs
  2. Click **Open in Browser** to verify it works
  3. Use the **Copy URL** button to share the address
</Accordion>
</AccordionGroup>

### Component Selection Not Working

<AccordionGroup>
<Accordion title="Verify Web Companion Installation">
  1. Check that `vibe-kanban-web-companion` is installed
  2. Ensure it's rendered in your app during development
  3. Look for the small companion widget in the bottom-right of your app

  <Tip>
  The companion only appears in development mode and when properly configured.
  </Tip>
</Accordion>

<Accordion title="Check Browser Console">
  Open browser developer tools and look for:
  - Web Companion initialisation messages
  - Any JavaScript errors that might prevent selection
  - PostMessage communication between the iframe and Vibe Kanban
</Accordion>

<Accordion title="Framework-Specific Issues">
  - **React**: Ensure the companion is rendered after React's reconciliation
  - **Next.js**: Verify SSR compatibility and client-side rendering
  - **Vue**: Check that the component is properly registered
</Accordion>
</AccordionGroup>

### Performance Considerations

<AccordionGroup>
<Accordion title="iframe Responsiveness">
  - Large applications may take time to load in the preview
  - Consider using development builds for faster loading
  - Monitor the Processes tab for build completion
</Accordion>

<Accordion title="Memory Usage">
  - Running multiple preview instances increases memory usage
  - Stop unused development servers when switching tasks
  - Use the **Stop Dev** button to clean up resources
</Accordion>
</AccordionGroup>
